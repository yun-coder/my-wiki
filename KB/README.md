# Knowledge Base - AI 知识库

> 基于 [Andrej Karpathy LLM Knowledge Base](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f) 架构的个人知识管理系统。

## 核心理念

传统 RAG 使用向量数据库检索 + 生成，但 Karpathy 提出了一个更优雅的方案：

> **不是检索后生成，而是边走边编译。**

LLM 自己维护一个 Markdown Wiki，在处理信息时不断编译、更新永久记忆。

## 架构概览

```
                    ┌─────────────────────┐
                    │   Sources           │
                    │  (Articles/Podcasts)│
                    └─────────┬───────────┘
                              │ 捕获
                              ↓
┌──────────────────────────────────────────────┐
│  Inbox ──→ Resources ──→ Knowledge ──→ Memory │
└──────────────────────────────────────────────┘
     ↑           原始笔记      编译        AI可用
```

## 目录结构

| 目录 | 用途 |
|------|------|
| `00_Inbox/` | 快速捕获，任何想法先丢这里 |
| `01_Daily/` | 每日笔记，日程、反思 |
| `02_Areas/` | 领域追踪：项目、目标、健康 |
| `03_Resources/` | 原始资料：文章、播客、书籍笔记 |
| `04_Knowledge/` | **核心**：AI 编译的永久笔记 |
| `05_Templates/` | 笔记模板 |

## 工作流程

### 1. 捕获 (Capture)
遇到任何有价值的信息 → 快速保存到 `03_Resources/`

### 2. 处理 (Process)
AI 阅读原始笔记 → 编译成原子卡片 → 存入 `04_Knowledge/`

### 3. 编译 (Compile)
AI 更新 `99_Memory.md` → 记忆作为系统提示词的一部分

### 4. 检索 (Retrieve)
对话时，AI 直接读取 Memory → 无需向量检索

## 为什么不用向量数据库？

- **个人规模足够**：~100 篇文章不需要复杂检索
- **更可解释**：Markdown 文件人类可直接阅读
- **AI 自我维护**：边聊边编译，不断完善
- **无额外基础设施**：纯文件系统

## 使用建议

1. **快速捕获**：有想法立刻记，不要追求完美
2. **定期整理**：AI 会帮你编译，但人工审核仍有价值
3. **持续对话**：把知识库当作第二大脑，不断提问和补充

## 技术栈

- **笔记工具**：Obsidian
- **存储**：GitHub (D:\GitHub\my-wiki)
- **AI 集成**：通过 OpenClaw 等工具调用
