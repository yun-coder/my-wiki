# Verification Before Completion - 完成前验证

## 核心原则

**声明工作完成但不验证是不诚实的。**

**证据在声明之前，总是这样。**

## 铁律

```
NO COMPLETION CLAIMS WITHOUT FRESH VERIFICATION EVIDENCE
```

如果还没在这个消息中运行验证命令，不能声称它通过了。

## 门控函数

```
在声称任何状态前：
1. 识别：什么命令能证明这个声明？
2. 运行：执行完整命令
3. 阅读：完整输出，检查退出码，计数失败
4. 验证：输出是否证实声明？
   - 如果否：陈述实际状态及证据
   - 如果是：陈述声明及证据
5. 只有这样：才能做声明
```

## 常见声明

| 声明 | 需要 | 不够 |
|------|------|------|
| 测试通过 | 测试命令输出：0 失败 | 之前的运行 |
| Linter 干净 | Linter 输出：0 错误 | 部分检查 |
| 构建成功 | 构建命令：exit 0 | Linter 通过 |
| Bug 修复 | 测试原始症状：通过 | 代码改了 |
