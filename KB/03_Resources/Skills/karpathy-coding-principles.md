# Karpathy AI 编程原则

> 灵感来源: Andrej Karpathy (OpenAI 联合创始人)

## 四大原则

### 1. Think Before Coding（编码前思考）
- 先理解需求，明确任何模糊点
- 提问而不是假设
- 先读懂现有代码再动手

### 2. Simplicity First（简洁第一）
- 优先用最简单的方案
- 不要为未来可能的需求提前优化
- 警惕"过度工程"的文化

### 3. Surgical Changes（手术式精准修改）
- 精准定位需要改的地方
- 每个改动都要有明确理由
- 分离关注点，每次只做一件事

### 4. Goal-Driven Execution（目标驱动执行）
- 开始前明确定义"完成"的标准
- 用验收标准驱动开发
- 达到目标就停止，不要过度优化

## 快速参考卡

| 原则 | 问自己的问题 |
|------|-------------|
| Think Before Coding | "我真的理解需求吗？" |
| Simplicity First | "有更简单的方案吗？" |
| Surgical Changes | "我只改了需要的吗？" |
| Goal-Driven Execution | "达到目标了吗？" |
